# User Intent Model

## Instructions

Transform the user's input into a structured intent model. Never discard the original wording.

## Required fields / sections

- Original request verbatim
- Interpreted business objective
- Problem being solved
- Desired outcome
- Likely users
- Known constraints
- Explicit requirements
- Implicit requirements
- Non-negotiable instructions
- Open decisions
- Potential contradictions
- Current-system context
- Definition of success

## Completeness checklist

- [ ] Original wording preserved verbatim
- [ ] Explicit vs implicit separated
- [ ] Non-negotiables captured
- [ ] Success is defined concretely
- [ ] JSON validates against user-intent.schema.json
