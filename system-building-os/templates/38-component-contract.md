# Component Contracts

## Instructions

For each significant reusable component define what it reflects (db/ai/api result), the requirement it represents, the API it consumes, the states it shows, and its error fallback.

## Required fields / sections

- Component id (CMP-###)
- Name/purpose
- Used in screens
- Requirement ref
- Consumes API
- State shown
- Reflects (db/ai/api result)
- Props/states
- Error fallback
- Accessibility
- Acceptance criteria / tests

## Completeness checklist

- [ ] Every product-logic component references a requirement
- [ ] Every data component names its API/AI source
- [ ] Error fallback defined
- [ ] Validates against component-contract.schema.json
